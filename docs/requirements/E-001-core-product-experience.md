# E-001 核心產品體驗建立

## 背景與問題
- 這份 Epic 用來承接產品第一版的核心體驗需求。
- 所有來自 kickoff、會議紀錄或使用者訪談的功能性需求，都應先進入這裡整理，再決定哪些能升級到 PRD。

## 目標與非目標

### 目標
- 提供產品第一版的需求母題與需求邊界。
- 讓初期需求有共同歸屬，避免直接把討論內容寫進 PRD。

### 非目標
- 本 Epic 不直接等於 PRD。
- 本 Epic 不直接定義技術設計或 implementation plan。

## 當前決策
- 第一版產品需求先由此 Epic 承接。
- 子需求編號格式固定為 `E-001-xxx`。
- 只有已定案、範圍具體、可驗收的子需求才可升級進 PRD。
- 目前先把「上傳音檔並取得逐字稿 / 字幕」視為可交付的 MVP 主流程。
- 影音轉音訊工具視為相鄰能力，但不作為本 Epic 的 MVP 驗收門檻。

## Phase 1
- 整理 kickoff 與初期討論中的核心需求。
- 收斂第一版使用者體驗與 scope。

## Phase 2
- 將已定案需求逐步升級到 PRD。
- 補齊驗收方式、限制條件與例外情境。

## Phase 3
- 將已完成與已驗收項目沉澱為穩定產品基線。

## Requirement List

### E-001-001 核心主流程可用
- 狀態：部分完成
- 類型：已定案
- 第一版主要使用者可上傳音檔，等待背景轉錄完成後取得逐字稿文字與 SRT 字幕。
- MVP 主流程至少包含：上傳、查詢處理狀態、取得 TXT / SRT、複製或下載結果。
- 繁體轉換屬於既有主流程上的輔助能力，可視為 MVP 加分項，但不是唯一驗收門檻。
- 目前 code 已有可用實作，已進入 PRD / TDD 對齊。

### E-001-002 首屏或入口體驗清楚
- 狀態：部分完成
- 類型：待釐清
- 使用者進入產品後，應該能快速理解產品目的與下一步行動。
- 尚未進入 PRD / TDD。
- 目前首頁已可直接上傳並開始轉錄，但產品說明、入口引導與主功能層次仍未完全收斂。

### E-001-003 第一版範圍邊界清楚
- 狀態：部分完成
- 類型：已定案
- 需要明確列出第一版必做與不做，避免 scope 膨脹。
- 目前先以「可取得逐字稿」作為 MVP 驗收基線。
- 影音轉音訊、會議記錄、更多模型設定或進階結果管理，不作為目前 MVP 完成判定的必要條件。
- 已進入 PRD，作為 MVP scope 定義的一部分。

### E-001-004 音檔轉錄後可生成會議記錄
- 狀態：進行中
- 類型：已定案
- 使用者在既有音檔轉錄流程完成後，可以直接產出結構化會議記錄。
- 第一版會議記錄固定包含：`摘要`、`重點討論`、`決議事項`、`未決議事項`、`待辦事項`。
- `未決議事項` 用來承接尚未拍板、仍待確認或需後續追決策的議題，不與 `待辦事項` 混用。
- 第一版整理稿匯出至少支援 `Markdown` 與 `DOCX`，且下載檔名需帶原始音檔名與時間戳，避免多場會議產生難以辨識的重名檔案。
- 會議記錄生成服務需支援 OpenAI-compatible API，部署目標可為真正的 OpenAI API 或 vLLM。
- 第一版沿用既有逐字稿結果，不新增獨立的音檔直出會議記錄流程。
- 目前前端 UI 與匯出流程已部分完成，但正式版本需改由 backend 負責整理稿生成，前端僅保留顯示、編修與下載。
- 會議記錄屬於主流程完成後的延伸產出，不取代 TXT / SRT 的核心交付。

## 未決問題
- 首頁是否需要更明確區分核心轉錄流程與相鄰工具。
- 影音轉音訊是否要納入同一個產品 Epic 的正式 MVP 範圍。

## 進入 PRD 條件
- 子需求已定案。
- 對應產品行為與驗收方向清楚。
- 無重大策略衝突或依賴未解。
